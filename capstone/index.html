Turning search demand into a human review queue

A transparent content-opportunity scorer ranks pages with substantial impressions, mid-page visibility, and under-captured clicks, so a reviewer can decide where title and metadata work may be worthwhile.

Mina Maged Zekry Gayid · Refresh / Content Opportunity Scoring lane · July 2026

Abstract

This project asks which content items deserve a first human discoverability review. It uses public-safe, pseudonymized daily search-performance fields from the FlyRank ML Internship warehouse and a deliberately simple score based on impressions and observed click-through rate. A rule baseline produces a ranked queue with a reason code and an action label; a compact regularized score is evaluated against the same held-out review slice as a directional comparison. The model adds a modest lift over the rule while preserving interpretable recommendations. This is decision support, not a claim about Google ranking causality or the impact of any content change.

Problem statement

Content teams cannot inspect every URL. The decision supported here is: which pages should a human review first for a possible title, meta-description, or query-intent mismatch? The output is intentionally an ordered review queue, never an automatic publishing system.

Methodology

Signals. The scorer uses same-day impressions, clicks, CTR, and average position. High impressions are a volume/quick-win signal; CTR relative to position is a snippet-opportunity signal. All are knowable at the review decision moment.

Baseline. score = impressions × (1 − CTR), restricted to pages with at least 100 impressions and average position 5–20. Every row receives the reason code HIGH_IMPRESSIONS_LOW_CTR and the action label Review title and meta description.

Comparison. The compact model is a regularized logistic score trained only on earlier grouped content histories, then checked on a later held-out review slice. Splits are grouped by content/client and ordered in time. A leakage check excludes future clicks and trend/label fields; the Week 3 deliberate-leak experiment showed why a future-derived field can create a perfect but invalid score.

Ranked recommendations

  1. Review title and meta description when high impressions meet a low CTR at positions 5–20.
  2. Check query intent and SERP features before changing copy; a low CTR may be rational.
  3. Check technical/indexing context when the page cannot plausibly satisfy the query.
  4. Monitor, do not edit if a one-day spike or position volatility explains the score.

The reproducible queue writes to work/outputs/baseline_action_score.csv locally and is intentionally excluded from the public repository.

Limitations & honest framing

Observed associations are not causal claims. Search demand, seasonality, query intent, snippets, SERP features, and content quality can all explain CTR. The score does not prove that refreshing a page changes Google performance. It is a directional, human-in-the-loop prioritization aid.

# Kill Your Darlings: Curate Your Images

## Final image set for the current portfolio

| Placement | Keeper | Why it belongs |
|---|---|---|
| Home hero / About | [Real professional headshot](https://raw.githubusercontent.com/minagayid/minagayid/main/assets/mina-headshot-suit.jpg) | A real photograph is the right choice where the subject is Mina; it establishes trust without pretending that an AI portrait is authentic. |
| Work proof | Current live portfolio: [minagayid.github.io](https://minagayid.github.io/) | The portfolio itself is the real capture of the work. Future case-study visuals should be cropped screenshots of actual clinic operations, dashboards, prototypes, or repositories—not generated substitutes. |
| Decorative layer | None for now | The site already has a clean, calm identity. It does not need a hero texture or stock-like AI illustration to carry the message. |

## Calls made deliberately

- **Real over AI:** I kept the existing real headshot because it represents the person behind the clinical and operational work.
- **Real over AI:** Future project visuals will be actual captures: a sanitized workflow, dashboard, prototype, or repository view. No patient information, clinic identifiers, or private operations data will appear.
- **Consistent set:** No generated image is currently needed. If a future connective visual is added, it should use the identity kit's restrained teal / cloud palette and never imitate a real clinic, patient, or work result.

## Rejection note

I rejected a generic AI "futuristic healthcare dashboard" hero image. It would look polished but would not prove that I built or used the system it depicts; it would also compete with the real headshot and make the portfolio less trustworthy. The quieter alternative—real portrait plus real work captures—better supports an evidence-led portfolio.

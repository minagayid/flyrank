# Three Roads

## Free stack options

1. **GitHub Pages + plain HTML/CSS** — fastest, free, almost no maintenance; no server-side features.
2. **Cloudflare Pages + Astro** — fast and scalable with a good content workflow; adds a build system and deployment settings.
3. **Vercel + Next.js** — strong previews and a future path to APIs; more framework complexity than this small project needs.

## Choice

I chose **GitHub Pages + plain HTML/CSS**. The project is a small portfolio proof, so the simplest stack is the most honest fit. It is easy to inspect, version, deploy, and maintain without a paid service.

- **Maintenance:** edit one HTML file, commit, and let GitHub Pages redeploy.
- **Display:** responsive static page, readable on phone and desktop.
- **Backend:** not yet needed. If forms, private data, or authentication become real requirements, I would add a small managed service rather than inventing one early.
